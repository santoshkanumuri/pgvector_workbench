## Getting Started

First, run the development server:

```bash
npm run dev
# or
yarn dev
# or
pnpm dev
# or
bun dev
```

Open [http://localhost:3011](http://localhost:3011) with your browser.

### New Multi-Session Flow
1. Register or login (credentials stored in backend metadata DB).
2. Create a Session (name + target PostgreSQL URL you want to explore).
3. Click Connect on a session (activates & initializes pool on-demand).
4. Browse tables, metadata, run searches.

Tokens & active session persist in localStorage (`db-look-auth`). Use Logout to clear.

You can start editing the page by modifying `app/page.tsx`. The page auto-updates as you edit the file.

## Deploy

Frontend (Next.js) and backend (FastAPI) can be deployed separately. Set the frontend API base URL in `src/lib/api.ts` or via environment injection for production builds.

Ensure environment variables for the backend (APP_JWT_SECRET, DATABASE_URL) are configured on the server.

The easiest way to deploy your Next.js app is to use the [Vercel Platform](https://vercel.com/new?utm_medium=default-template&filter=next.js&utm_source=create-next-app&utm_campaign=create-next-app-readme) from the creators of Next.js.
